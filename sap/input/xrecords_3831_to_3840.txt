--- Record 3834 ---
Component: Cloud Identity Services
Environment: Cloud Foundry,Kyma,Neo
Title: Refresh Token Flow of OIDC Protocol Restricted to Validity of Web Session
Description: As announced 2023-08-16, with this change, the service couples the validity of refresh tokens to the session timeout. Refresh tokens expire with the user session, unless you add the offline_access scope.  Action: Check if you define a refresh token validity for your applications longer than 12 h:      Yes: Ensure that you decouple the refresh token from the user session with the offline_access scope.  For more information, see Token Policy Configuration for Applications.      No: Nothing to do.
