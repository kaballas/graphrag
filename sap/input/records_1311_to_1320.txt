Title: Destination Service - Destination Chaining
Description: The Destination service now provides a new feature that lets you invoke multiple destinations with a single Find a Destination request, where each targeted destination is used to produce the input for the invocation of the next one. The feature is called Destination Chaining.   With the initial release, this feature can only be used as a predefined destination chain to generate and consume IAS (Identity Authentication service)-signed SAML assertions.  For more information, see Destination Chaining.
